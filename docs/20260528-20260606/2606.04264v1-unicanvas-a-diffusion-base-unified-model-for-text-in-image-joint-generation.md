## 摘要
近年来，在单一架构内同时处理多模态理解与生成的统一视觉-语言模型取得了显著进展。尽管自回归VLM能够跨模态推理，但无法生成高质量图像。相比之下，扩散模型能生成逼真的视觉内容，却难以生成连贯的文字，这使得开发一个能无缝处理视觉和文本生成的统一模型充满挑战。最近的研究表明，语言可以有效地嵌入视觉表示中，使模型能够直接从图像推理文本语义。为此，我们提出UniCanvas，这是首次尝试通过文本-图像生成来统一扩散模型以生成交错多模态内容。扩散模型天然地捕捉共享像素画布上的变换，可视为视觉变化的世界模型。该模型不生成离散文本标记，而是学习将语言表示为图像内的视觉模式，利用其固有的多模态嵌入空间。这种设计使得模型在图像合成过程中能够在单一像素画布上自然地“绘制”文本，实现无缝的多模态生成。实验表明，UniCanvas相较于先前的统一模型提升了性能，将基于扩散模型的文本-图像生成定位为一种有前途的统一多模态生成范式。

## Abstract
Recent years have seen remarkable progress in unified vision-language models handling both multimodal understanding and generation within a single architecture. While autoregressive VLMs can reason across modalities, they fail to generate high-quality images. In contrast, diffusion models produce photorealistic visuals yet struggle to generate coherent text, making it challenging to develop a single unified model that can seamlessly handle both visual and text generation. Recent advances suggest that language can be effectively embedded within visual representations, allowing models to reason about textual semantics directly from images. To this end, we propose UniCanvas, a first attempt that unifies diffusion models to generate interleaved multimodal contents through text-in-image generation. Diffusion models naturally capture transformations on a shared pixel canvas, which can be viewed as world models of visual change. Instead of producing discrete text tokens, the model learns to represent language as visual patterns inside images, leveraging its inherent multimodal embedding space. This design allows the model to "draw" text naturally within a single pixel canvas during image synthesis, achieving seamless multimodal generation. Experiments demonstrate that UniCanvas improves performance over previous unified models, positioning text-in-image generation with diffusion models as a promising unified multimodal generation paradigm.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有统一多模态模型存在三个关键鸿沟：① 目标鸿沟（文本用下一Token预测损失，图像用重构或去噪损失）；② 表示鸿沟（文本在符号空间，图像在像素空间）；③ 推理鸿沟（推理主要在语言头中，与图像生成器脱耦）。结果导致自回归VLM能推理但图像质量差，扩散模型能生成逼真图像但文本生成能力弱，难以在单一架构中无缝处理视觉与文本的联合生成。
- **背景**：近年已有多种统一模型尝试（如Anole、MVoT、MMaDA、BAGEL等），但它们仍将文本和图像视为分离任务，分别使用独立头进行解码。UniCanvas提出一种新范式：在共享的像素画布上统一表达文本和图像，让模型在图像合成过程中自然“绘制”文本，实现文本-图像联合生成。
- **整体含义**：首次证明扩散模型可以在单一像素画布上通过文本-图像生成实现多模态统一，为跨模态推理和生成提供新方向。

## 2. 论文提出的方法论

### 核心思想
UniCanvas基于扩散模型，在共享的像素画布（潜在空间）上同时表示文本和图像。文本不再作为离散Token，而是作为光学字符模式直接渲染在画布上。模型学习在画布上“写”文本并更新场景状态，实现多步推理。

### 关键技术细节
1. **文本-图像生成（Text-in-Image Generation）**：采用流匹配（Flow Matching）框架。给定源图像 \(x\) 和目标文本增强图像 \(x^{\text{text}}_0\)，采样 \(t \sim U[0,1]\)，构建线性插值 \(x_t = (1-t)x + t x^{\text{text}}_0\)，训练速度网络 \(v_\theta(x_t, t, c_{\text{reason}})\) 预测真实速度 \(x^{\text{text}}_0 - x\)。训练损失：
   \[
   \mathcal{L}^{\text{text}}_{\text{FM}} = \mathbb{E}_{t, x, x^{\text{text}}_0} \left[ \| v_\theta(x_t, t, c_{\text{reason}}) - (x^{\text{text}}_0 - x) \|^2 \right]
   \]
   推理时从输入图像开始，用ODE求解器（如Euler）逆向积分得到文本渲染画布。

2. **两阶段生成**：分为视觉推理（Stage 1）和视觉合成（Stage 2），共享同一扩散主干。
   - **Stage 1（视觉推理）**：基于推理提示 \(c_{\text{reason}}\)，将语义文本“写”到画布上，得到推理增强画布 \(\hat{x}^{\text{text}}_0\)。
   - **Stage 2（视觉合成）**：基于执行提示 \(c_{\text{exec}}\)，将推理画布变换为下一状态图像 \(\hat{x}^{\text{state}}_0\)。训练损失类似：
     \[
     \mathcal{L}^{\text{state}}_{\text{FM}} = \mathbb{E}_{t, \hat{x}^{\text{text}}_0, x^{\text{state}}_0} \left[ \| v_\theta(x_t, t, c_{\text{exec}}) - (x^{\text{state}}_0 - \hat{x}^{\text{text}}_0) \|^2 \right]
     \]
   - 多步任务：将 \(\hat{x}^{\text{state}}_0\) 作为新的输入，重复两阶段过程。

3. **CLIP对齐正则化**：使用冻结的CLIP视觉编码器 \(E^{\text{vis}}_\phi\) 和文本编码器 \(E^{\text{txt}}_\phi\)，对推理画布 \(\hat{x}^{\text{text}}_0\) 和对应推理提示 \(c_{\text{reason}}\) 计算对比损失，增强语义对齐。总损失：
   \[
   \mathcal{L}_{\text{total}} = \mathcal{L}^{\text{text}}_{\text{FM}} + \mathcal{L}^{\text{state}}_{\text{FM}} + \lambda_{\text{CLIP}} \mathcal{L}_{\text{CLIP}}
   \]
   其中 \(\lambda_{\text{CLIP}}=0.1\)。

4. **预训练阶段**：使用VQA数据（LLaVA-Instruct-150K）和通用图像编辑数据（OmniEdit-Filtered-1.2M），将答案渲染为图像中的光学文本，并添加LPIPS感知损失（\(L_{\text{perc}} = d_{\text{lpips}}(\text{crop}(\hat{x}^{\text{text}}_0), \text{crop}(x^{\text{text}}_0))\)），以建立基础的文本渲染能力。

## 8. 不足与局限

1. **长序列生成不稳定**：长序列中视觉合成误差会累积（图6），导致最终状态偏离真实轨迹；文本渲染也可能出现噪点或字符幻觉（图7）。
2. **通用推理能力弱**：在开放式视觉问答上落后于自回归VLM，主要因为监督有限（仅使用LLaVA数据）且模型更擅长短句而非多句解释。
3. **推理速度慢**：两阶段扩散（每步40步×2）比自回归模型慢（相对时间4.6 vs 1.0），但优于其他扩散顺序基线。
4. **实验范围有限**：通用推理仅200样本，未在更大基准（如VQA v2、MMBench）上评估；未测试真实世界图像编辑任务。
5. **对预训练数据依赖强**：缺乏文本-图像预训练时性能崩溃（平均0.23），需要大量特定数据训练文本渲染能力。
6. **仅支持固定分辨率**：基于Qwen-Image-Edit，可能不适用于任意尺寸图像。
