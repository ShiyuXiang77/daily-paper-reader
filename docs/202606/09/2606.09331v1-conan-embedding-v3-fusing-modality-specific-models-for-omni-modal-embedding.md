## 摘要
全模态检索旨在为文本、图像、视频、文档和音频输入提供统一的嵌入空间，但由于这些模态在数据分布、架构和优化动态方面的差异，构建这样一个统一的检索器颇具挑战。本文提出Conan-embedding-v3，一种用于全模态检索的解耦-融合-恢复框架。Conan-embedding-v3首先独立训练模态专有模型，然后将它们的任务向量融合到一个单一的密集骨干网络中，我们称此策略为解耦专有模型融合。我们证明这种融合组合了视觉、视频和文档检索能力，但也暴露了基于投影器的模态的失败模式：当通过外部编码器和投影器连接音频时，融合骨干网络会使投影器与音频专有骨干网络之间的校准失效，导致尽管原封不动地复制了所有音频专有模块，音频检索性能却大幅下降。我们将此失败称为投影器漂移。为修复此问题，Conan-embedding-v3应用投影器恢复（即保持骨干网络冻结的情况下对投影器进行全参数微调），随后进行平衡的多模态重放。最终模型在单一骨干网络中支持这些检索路径，在MMEB上获得74.9分，同时在包含30个任务的MAEB音频套件上达到55.61分。

## Abstract
Omni-modal retrieval promises a single embedding space for text, image, video, document, and audio inputs, but building such a unified retriever is difficult since these modalities differ in data distribution, architecture, and optimization dynamics. In this work, we present Conan-embedding-v3, a decouple--fuse--recover framework for omni-modal retrieval. Conan-embedding-v3 first trains modality specialists independently and fuses their task vectors into a single dense backbone, a strategy we call Decoupled Specialist Fusion. We show that this fusion composes visual, video, and document retrieval capabilities, but also exposes a failure mode for projector-based modalities: when audio is attached through an external encoder and projector, fusing the backbone leaves the projector calibrated to the audio-specialist backbone, causing a large audio retrieval regression despite copying all audio-specific modules unchanged. We call this failure Projector Drift. To repair it, Conan-embedding-v3 applies Projector Recovery (i.e., full-parameter fine-tuning of the projector while keeping the backbone frozen) followed by balanced multi-modal rehearsal. The resulting model supports these retrieval pathways in one backbone, achieving 74.9 scores on MMEB while obtaining 55.61 on the 30-task MAEB audio suite.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：构建统一的文本、图像、视频、文档和音频检索嵌入模型时，不同模态在数据分布、架构和优化目标上存在显著差异，直接联合训练会导致跨模态优化冲突（“跷跷板效应”），一个模态的性能提升往往以牺牲其他模态为代价。特别地，音频模态通常以“嫁接”方式（外部编码器+投影器）接入视觉语言骨干网络，融合后其投影器与骨干网络之间会出现严重失配，导致音频检索性能大幅下降。
- **研究动机**：希望得到一个单一的、支持所有主流模态检索的嵌入模型，同时避免传统联合训练中的优化冲突和模态退化问题。
- **整体含义**：提出一种“解耦-融合-恢复”框架，先独立训练各模态专家，再在参数空间融合，最后修复音频投影器的漂移，从而在单一骨干网络中同时保留强视觉和强音频检索能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想：Decouple-Fuse-Recovery
将跨模态能力获取与参数融合解耦，通过独立优化各模态专家，再以任务向量融合构建统一骨干，最后针对嫁接模态的接口失配进行后修复。

### 关键技术细节

1. **阶段一：解耦专有模型训练 (Decoupled Specialist Training)**
   - 从同一基础模型 θ₀ 出发，分别训练图像（I）、视频（V）、视觉文档（D）、音频（A）四个专家模型 θ<sub>m</sub>，每个专家仅使用其对应模态的检索对数据，避免跨模态冲突。
   - 音频专家额外嫁接外部音频编码器和两层MLP投影器，将音频特征映射至骨干网络隐藏维度。
   - 所有专家均使用LoRA适配器，训练后合并为完整稠密参数，以便进行参数空间操作。

2. **阶段二：任务向量融合 (Task-Vector Fusion)**
   - 对共享骨干参数 K<sub>shared</sub>，定义每个专家的任务向量 Δ<sub>m</sub> = θ<sub>m</sub> - θ₀。
   - 融合后的骨干：θ<sub>merge</sub> = θ₀ + Σ α<sub>m</sub> Δ<sub>m</sub>，其中融合系数通过网格搜索确定（多路C配置：α<sub>I</sub>=α<sub>V</sub>=α<sub>D</sub>=0.3，α<sub>A</sub>=0.5）。
   - 音频专有模块（编码器、投影器、特殊token）直接复制自音频专家。
   - 融合后产生“投影器漂移”问题：投影器原与音频专家骨干对齐，现连接到融合后的不同骨干，导致输出空间严重偏移。

3. **阶段三：恢复训练 (Recovery Training)**
   - **投影器恢复**：冻结融合后的骨干和音频编码器，仅对轻量级音频投影器（约19M参数）进行全参数微调，基于音频-文本检索对重新校准接口。
   - **平衡多模态重放**：从投影器恢复基点出发，用低学习率对骨干和视觉编码器进行轻量级LoRA微调，混合音频、图像、视频、文档数据（约2000步），以保持并协调所有检索通路。

## 6. 论文的主要结论与发现

1. **解耦专家融合有效避免了跨模态优化冲突**：独立训练各专家后参数融合，图像、视频、文档的预融合上限（80.3/67.2/82.0）显著高于联合训练的PCGrad（72.3/57.7/70.4）。
2. **投影器漂移是嫁接模态融合的关键失败模式**：融合骨干后，音频投影器输出空间与骨干不再匹配，导致R@1从92%暴跌至56%；该漂移在深层Transformer层中更严重。
3. **投影器恢复+平衡重放能有效修复接口**：仅微调投影器即可恢复音频性能至55.82（MAEB），平衡重放进一步提升视觉性能至77.2/65.1/79.0（MMEB）而不损害音频。
4. **最终模型性能**：MMEB总分74.96，MAEB总分55.61，超过所有同期全模态嵌入模型（如LCO-Embedding-Omni-7B: 53.54, jina-embeddings-v5-omni-small: 50.41），在视觉领域也接近顶尖视觉专用模型（Qwen3-VL-Embedding-8B: 77.82）。
