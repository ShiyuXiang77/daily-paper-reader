## 摘要
多模态大语言模型（MLLMs）面临显著的推理瓶颈，因为长视觉标记序列上的自注意力计算成本是二次的。然而，我们发现了当前架构中的一个关键低效问题：视觉注意力饱和。我们的分析表明，视觉标记在早期层中快速建立其空间结构和模态内关系，使得深层中的视觉-视觉自注意力在计算上变得冗余。相反，这些层中的前馈网络（FFNs）对于将视觉特征投影到不断演变的文本语义空间仍然至关重要。利用这一见解，我们提出了Visual-Skip（V-Skip），一种无训练的推理范式，它将空间交互与语义演化解耦。V-Skip不丢弃标记，而是通过选择性跳过饱和的视觉自注意力模块来施加块级结构稀疏性。此外，识别到不同的下游任务需要不同的推理深度，V-Skip采用轻量级的少量样本校准来动态路由任务最优的稀疏路径。大量实验表明，V-Skip有效绕过冗余的视觉注意力以实现块级稀疏性，在各种MLLMs上保持94.16%至100.31%的性能保留。最终，我们证明了为了更有效地推理，模型不需要丢弃它们看到的内容——它们只需在正确的深度上“少看”。

## Abstract
Multimodal Large Language Models (MLLMs) face a significant inference bottleneck due to the quadratic computational cost of self-attention over long visual token sequences. However, we identify a critical inefficiency in current architectures: Visual Attention Saturation. Our analysis reveals that visual tokens rapidly establish their spatial structure and intra-modal relationships in early layers, rendering visual-to-visual self-attention in deeper layers computationally redundant. Conversely, Feed-Forward Networks (FFNs) in these layers remain essential for projecting visual features into the evolving textual semantic space. Leveraging this insight, we present Visual-Skip (V-Skip), a training-free inference paradigm that decouples spatial interaction from semantic evolution. Rather than discarding tokens, V-Skip imposes block-wise structured sparsity by selectively bypassing saturated visual self-attention modules. Furthermore, recognizing that varying downstream tasks demand distinct reasoning depths, V-Skip employs a lightweight, few-shot calibration to dynamically route the task-optimal sparsity path. Extensive experiments demonstrate that V-Skip effectively bypasses redundant vision attention to achieve block-wise sparsity, maintaining a 94.16% to 100.31% performance retention across diverse MLLMs. Ultimately, we prove that to reason more effectively, models do not need to discard what they see -- they simply need to "look less" at the right depth.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：多模态大语言模型（MLLMs）在处理高分辨率图像时，会将视觉内容编码为长序列视觉 token（例如从 576 个扩展到 2800+ 个）。Transformer 自注意力的计算复杂度是序列长度的平方，导致推理阶段（特别是预填充阶段）延迟和内存开销剧增。
- **现有方法的局限**：当前主流的加速方法包括 token 剪枝（如 VTW）和整层跳过（如 ShortV）。但这些方法都是“破坏性”的：剪枝永久丢失细粒度视觉细节，整层跳过同时丢弃了关键的前馈网络（FFN）变换，导致语义对齐错位和物体幻觉。
- **核心问题**：是否存在一种更精细的结构性冗余利用方式，既能大幅降低计算量，又不损害模型的感知和推理能力？
- **整体含义**：论文提出“少看多思”理念——模型不需要丢弃看到的视觉信息，只需在正确的深度减少冗余的视觉‑视觉自注意力计算，同时保留必要的语义演化（FFN），从而实现高效且可信的推理。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：发现 MLLMs 中视觉处理存在功能解耦——早期层视觉自注意力用于建立空间结构（高度需要），深层层视觉自注意力逐渐饱和（冗余），但深层 FFN 对于将视觉特征投影到不断演变的文本语义空间仍然关键。V-Skip 选择性跳过深层饱和的视觉自注意力模块，保留 FFN，实现块级结构稀疏性。
- **关键技术细节**：
  1. **Visual Information Gain (VIG) 度量**：通过比较完整模型和跳过某层视觉自注意力（保留 FFN）后的输出概率分布之间的 KL 散度，量化该层视觉自注意力的功能重要性。VIG 越小，说明该层视觉自注意力越饱和（冗余）。
  2. **任务感知的少样本校准**：从下游任务数据集中随机采样 20 个图像‑文本对，计算每层的 VIG 值，选择 Top‑N 个最低 VIG 的层作为跳过层。由于不同任务需要的推理深度不同，这种校准机制能自动适应任务特点。
  3. **块级注意力跳过机制**：在选定的跳过层中，修改注意力掩码，使视觉 query token 不参与对其他视觉 token 的注意力计算（残差连接直接传递），而文本 token 仍然保持对全序列的完整注意力。FFN 对所有 token 正常执行，保证视觉特征的语义演化。
  4. **复杂度优化**：跳过层中，原始 O(N_v² + N_v·N_t + N_t²) 的复杂度降至 O(N_t·N_v + N_t²)，由于 N_v >> N_t，消除了主导的二次项 N_v²，理论加速比约为 N_v/N_t + 1。

## 6. 主要结论与发现

- **视觉注意力饱和现象普遍存在**：深层层（约后 60% 层）的视觉‑视觉自注意力功能冗余，FFN 仍保持关键作用。
- **V-Skip 实现近无损加速**：在 LLaVA‑1.5‑7B 上达到 100.31% 平均性能保留（甚至略超原始模型），LLaVA‑NeXT‑7B 上 98.42%，Qwen2.5‑VL‑7B 上 94.16%（通过降低稀疏预算可提升至 98.72%）。
- **优于破坏性方法**：V-Skip 在所有架构上均显著优于 VTW 和 ShortV，尤其避免了物体幻觉和图像细节丢失（OCR 任务从 VTW 的 5.1 提升至 31.6）。
- **即插即用增强**：与 FastV 结合进一步压缩计算且恢复推理能力，证明 V-Skip 可作为通用加速组件。
- **任务自适应必要性**：不同任务需要不同的稀疏路径，轻量少样本校准可以高效捕捉任务特异性。

## 7. 优点

- **无训练、零成本**：不需要额外训练或微调，仅需一次轻量校准（~2 分钟），可直接应用于预训练模型。
- **非破坏性**：保留所有视觉 token 完整性，避免信息永久丢失；保留 FFN 确保语义连续对齐。
- **任务自适应**：通过 VIG 度量 + 少量样本校准，自动适应不同下游任务的推理深度需求，优于统一固定策略。
- **理论清晰且可解释**：基于对注意力饱和现象的深入分析，设计简洁，有明确的理论动机。
- **通用性**：在 LLaVA 系列和 Qwen2.5‑VL 两类主流架构上均有效，且可与 token 剪枝方法互补。
- **缓解幻觉**：在 POPE 和复杂推理任务上表现优于剪枝方法，表明保持完整视觉上下文有助于减少物体幻觉。

## 8. 不足与局限

- **校准依赖下游任务**：虽然少样本开销低，但仍需为每个新任务或领域进行一次校准，无法做到完全零样本通用。文中提出的 V-Skip*（统一使用 MME 校准的配置）性能略低于任务特定校准（99.55% vs 100.31%）。
- **极端稀疏时性能下降**：当跳过层数过多（如 N=32）时，性能显著下降（91.08%），说明某些深层视觉自注意力对特定输出仍有贡献，饱和是相对而非绝对。
- **Qwen2.5-VL 保留率较低**：默认 N=7 时仅 94.16%，需要重校准到 N=4 才能接近 98.72%，且仍低于 LLaVA 系列的近无损水平，说明不同架构的饱和模式差异大，需针对性调参。
- **实验覆盖有限**：仅在 7B/13B 规模上验证，未测试更大模型（如 70B+）；也只对比了无训练方法，未与基于蒸馏或结构化剪枝的训练方法比较。
- **未报告端到端推理吞吐量**：只报告了理论 TFLOPs 和单层延迟，缺乏完整批处理、KV-cache 优化等实际部署场景的吞吐量数据。
- **评估框架确定性**：论文承认 lmms-eval 是确定性的，但未提供多次运行的统计显著性检验（如标准差），虽然消融中展示了种子稳定性，但主表结果缺少置信区间。
