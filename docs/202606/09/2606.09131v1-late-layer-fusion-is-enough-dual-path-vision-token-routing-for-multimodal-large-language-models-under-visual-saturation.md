## 摘要
多模态大语言模型（MLLMs）通常继承为单模态文本建模设计的深层、对称Transformer骨架，并对图像和语言令牌均匀应用相同计算。这种设计忽略了一个关键的模态不对称性：图像和文本令牌在信息密度、冗余性和所需推理深度上存在显著差异。通过对LLaVA-1.5的逐层分析，我们观察到视觉令牌在中间层趋于饱和。具体而言，文本到图像的注意力从第0层的0.68下降到第4层的0.07，并在第18层后稳定在0.04附近，而文本令牌则继续受益于深层语义处理。这些发现表明，架构对称性与深度异步模态演化之间存在不匹配，导致视觉计算冗余以及在深层任务特定适应期间感知表示的潜在漂移。基于此，我们提出双路径视觉令牌路由（DPVR），一种用于高效MLLMs的模态异步路由框架。其核心实例化DPVR-LF（后期层融合）在饱和点将视觉令牌路由到单层可训练的侧分支，执行跳过深层堆栈中图像位置的十三层纯文本前向传播，并仅在最终层重新融合视觉和文本流。仅需约3%的可训练参数，DPVR-LF在标准基准上保持了有竞争力的多模态性能，同时减少了深层Transformer堆栈中的视觉计算。该结果挑战了视觉令牌必须遍历所有深层语言模型层的传统假设，并表明单个后期融合层足以在LLaVA风格的MLLMs中维持强大的感知能力。

## Abstract
Multimodal large language models (MLLMs) commonly inherit the deep, symmetric Transformer backbone designed for unimodal text modeling, and apply the same computation uniformly to image and language tokens. This design overlooks a key modality asymmetry: image and text tokens differ substantially in information density, redundancy, and required reasoning depth. Through a layer-wise analysis of LLaVA-1.5, we observe that vision tokens tend to saturate in the middle layers. Specifically, text-to-image attention decreases from 0.68 at layer 0 to 0.07 by layer 4, and stabilizes near 0.04 after layer 18, whereas text tokens continue to benefit from deep semantic processing. These findings suggest a mismatch between architectural symmetry and depth-asynchronous modality evolution, resulting in redundant visual computation and possible drift in perceptual representations during deep task-specific adaptation. Motivated by this, we propose Dual-Path Vision Token Routing (DPVR), a modality-asymmetric routing framework for efficient MLLMs. Its core instantiation, DPVR-LF (Late-Layer Fusion), routes vision tokens at the saturation point into a one-layer trainable side branch, runs a thirteen-layer text-only forward that skips image positions in the deep stack, and re-fuses the visual and textual streams only at the final layer. With approximately 3% trainable parameters, DPVR-LF preserves competitive multimodal performance on standard benchmarks while reducing visual computation in the deep Transformer stack. The results challenge the conventional assumption that vision tokens must traverse all deep language-model layers, and indicate that a single late fusion layer can be sufficient for maintaining strong perceptual competence in LLaVA-style MLLMs.

这些现象表明**架构对称性与模态深度异步演化之间存在结构性错配**：视觉 token 在中间层即达到饱和，深层计算对其近乎冗余，甚至可能在任务适应过程中引入感知漂移。基于此，作者提出一种模态异步路由框架 DPVR（Dual-Path Vision Token Routing），旨在减少深层视觉计算同时保持表达能力。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

**核心思想**：在视觉饱和点将视觉 token 从主路由中分离，经由一条浅层可训练侧分支处理，文本 token 则独立完成深层推理，最终在最后一个层重新融合两路信息。

**关键技术细节**：

- **饱和点选取**：基于实证分析，选取视觉饱和点 \(s = 18\)（7B 模型）或 \(s = 28\)（13B 模型）。  
- **算法流程（文字说明）**：
  1. **共享浅层（冻结）**：从层 0 到层 \(s-1\)，视觉和文本 token 一起通过完整的 LlamaDecoderLayer，参数冻结。  
  2. **侧分支（可训练）**：在层 \(s\)，将视觉 token 路由至一个单层可训练的 LlamaDecoderLayer 进行抽象（约 202M 参数，占总参数 3%）。

  4. **最终层融合**：在最后一层 \(L-1\)，将侧分支输出的视觉表示与深层文本表示重新拼接，执行一次完整的注意力计算，实现图文融合。

- **梯度流动**：由于 LLaVA-1.5 的标签只在 assistant 响应位置非 -100，若没有最终融合，侧分支梯度恒为零（DPVR-LF-ideal 不可训练）。通过最终融合层，文本 query 可以 attend 到侧分支输出的视觉 KV，从而建立梯度回溯路径（公式 (1)）。

### 6. 论文的主要结论与发现

1. **视觉 token 不需要遍历所有深层语言模型层**：单个后期融合层足以维持 LLaVA 风格 MLLM 的感知能力。  
2. **仅需约 3% 可训练参数（202M）**，DPVR-LF 在 8 个基准上匹配或超越全微调，同时实现 **25–30% 前向 FLOPs 节约**（实测延迟降低 28.0%）。  
3. **饱和点是鲁棒的结构性属性**：分点选择在较宽范围内（如 7B 的 18–24）不影响性能。  
4. **视觉深度饱和跨模型大小一致**：7B 和 13B 均显示单层侧分支最优。  
5. **延迟节约完全来自预填充阶段**，解码阶段因未优化 KV 缓存而暂未获益。

### 7. 优点

- **方法简洁高效**：仅修改路由逻辑，不改变视觉编码器或投影器，可即插即用于现有 LLaVA 检查点。  
- **动机扎实**：基于逐层定量分析（注意力质量、余弦相似度、logit lens）而非直觉，发现具有可复现性。  
- **实验全面**：覆盖多模型大小、多硬件、多消融；包括跨硬件延迟验证和文本长度扫掠，增强了结论的泛化性。  
- **诚实披露局限性**：明确报告弱项（如 BLINK 和 MMBench-CN 的性能下降）、解码阶段假设未实现等。  
- **提供理论解释**：证明 DPVR-LF-ideal 不可训练，并展示最终融合层的梯度回溯机制。

### 8. 不足与局限

- **多图像与跨语言任务敏感**：DPVR-LF 在 BLINK（−2.0 pp）和 MMBench-CN（−1.9 pp）上表现下降，说明当前单层融合对多图像关系推理和跨语言对齐支撑不足。  
- **骨干验证有限**：仅在 LLaVA-1.5（7B/13B）上完整训练，仅在 LLaVA-Next Mistral 上简单前向测试，未在 Qwen-VL-2 等新骨干上全面训练。  
- **梯度信号稀疏**：仅有 1 条注意力梯度路径（vs. DPVR-PC 的 14 条），需 2× 学习率补偿；更激进的 DPDR-LF-ideal 完全不可训练。  
- **解码阶段未优化**：当前实现中解码时因 KV 缓存形状不一致而退化到无缓存全上下文前向，延迟节约仅见于预填充阶段。  
- **随机种子控制不彻底**：三个种子共享同一数据 shuffle 顺序，只能反映初始化方差，不能完全代表训练方差。  
- **计算 FLOPs 节约的理论估计** 依赖于经验校准参数 \(\rho = 0.70\)，该参数在长文本场景下可能变化（实测在 \(T_{txt}=1024\) 时节约降至 8%）。
